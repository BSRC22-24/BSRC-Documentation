VLAN

Lien Trunks

[ ] == élément variable (Interface, IP, Nom, etc….), Ne pas inclure "[" et "]" dans la commande CLI

{ } == option de commande Ne pas inclure "{" et "}" dans la commande CLI

Activer le mode Trunk

Activer le mode Trunk sur les port du switch :

  • Sélection d’interface groupée : interface range [fastEthernet 0/23 - 24]

ici Sélectionne les interface 23 et 24

  • Activation des interfaces en mode Trunk : switchport mode trunk

Domaine VTP

Le domaine VTP permet de créer les VLAN automatiquement sur les switch avec le mode Trunk Activé. Pour fonctionner le switch doit être en mode ==Serveur== (Mode par défaut).

Pour changer de mode : vtp mode {server}. =={server}{client}, {transparent}==

  • Créer le domaine VTP : vtp domain [NomDuDomaine]
  • Activer le Pruning (Permet de restreindre la propagation des informations de VLAN aux liens nécessaires, réduisant ainsi la charge réseau inutile) : vtp pruning
  • Créer un mot de passe pour le domaine : vtp password [Password]
  • Changer de Version de VTP (à faire sur chaque switch) : vtp version {2} =={1}{2}{3}==
  • Afficher le mot de passe du domaine : show vtp password
  • Afficher l’état du domaine VTP : show vtp status

Connecter un Switch en mode ==Client==, ne pas oublier de définir le même mot de passe que le domaine vtp password [Password]

  • Afficher les VLAN : show vlan brief

Un fois le réseau VTP créer, on peut créer les différents VLAN qui seront créer sur tous les switch relier en Trunk.

(en mode conf t)

  • Créer un VLAN : VLAN [10]
  • Nommé le VLAN : name [Nom]

Attribution des Interfaces au VLAN

Un fois les VLAN Créer nous pouvons attribué les interfaces leur correspondant :

==(à faire sur chaque switch correspondant au bon VLAN)==

(en mode conf t)

  • Sélection d’interface groupée : interface range [fastEthernet 0/1 - 20]

Ici Sélectionne les interface 1 a 20

  • Activation des interfaces en mode "Access": switchport mode access

  • Attribution des interfaces au VLAN: switchport access vlan [10]

Si Trunk ne veut pas se mettre : switchport trunk encapsulation dot1q

Sous interface Routeur :

Un fois les VLAN créer, nous allons créer des sous interfaces dans la routeur pour pouvoir gérer les VLAN via 1 seule interface physique du routeur.

(en mode conf t - Aucune adresse ip doit être attribuer a l’interface physique)

  • Création d’un sous interface : interface fastEthernet [0/0.10]

  • Attribution du VLAN : encapsulation dot1Q [10] ([10] = pour VLAN 10 par exemple.)

  • attribution de l’ip : ip address [172.16.10.254 255.255.255.0] (= [ ip + Subnet Mask])

A faire pour chaque VLAN…

  • Vérifier les sous interfaces : show ip interface brief
