Commandes CISCO

Vocabulaire

  • Name = Nom (ex : R1 )
  • Password = Mot De Passe (ex: cisco )
  • Int = Interface (ex: Fa0/0 )
  • Ints = Interfaces (range, ex: Fa0/0-24 )
  • Ipv4 = Adresse IPV4 (ex: 192.168.1.254 )
  • Mask = Masque SR (ex: 255.255.255.0 )
  • Network = Adresse Réseau (ex: 192.168.1.0 )
  • Cidr = CIDR (ex: /24 )
  • Vlan_id = ID du vlan (ex: 10 )
  • Vlan_name = Nom du vlan (ex : SALES )

P.1 - Routeurs

Liste des différentes commandes permettant la configuration d'un routeur CISCO dans Packet Tracer.

Pour entrer dans le monde config: en (mode Privilège) puis conf t (mode Configuration) & wr pour save.

P1.1 - Configuration basique

  • Changer l'Hostname : hostname <name>
Configurer une interface : interface <int> ( int ) * **Ajout IP** : `ip address ` _( ip add )_ * **Activer l'interface** : `no shutdown` ( _no shut_ ) * **Sortir** : `exit`
  • Sortir du mode configuration : exit ( CTRL+Z )
  • Sauvegarder la configuration : copy running-config startup-config ( wr )
  • Voir la configuration actuelle : show running-config ( sh r )

P1.2 - Routage statique

  • Ajouter une route : ip route <network> <mask> <nexthop>
  • Supprimer une route : no ip route <network> <mask> <nexthop>

P1.3 - DHCP VLAN

Création POOL pour VLAN : ip dhcp pool VLAN * **Définir le réseau** : `network ` * **Définir le routeur** : `default-router ` * **Définir le serveur DNS** : `dns-server `

P1.4 - Sécurité

  • Mettre mot de passe : line console 0 & password <password> & login

P1.5 - Divers

  • Transmettre requête DHCP : ip helper-address <ipv4>
  • Voir liste client DHCP : sh ip dhcp binding

P.2 - Switchs

Liste des différentes commandes permettant la configuration d'un switch CISCO dans Packet Tracer.

Pour entrer dans le monde config: en (mode Privilège) puis conf t (mode Configuration) & wr pour save.

P2.1 - Configuration d'un VLAN dans un SWITCH

Configuration d'une range en mode TRUNK : int range <ints> * **Mode TRUNK** : `sw mode trunk` ( _sw mo tr_ )
Configuration VTP : * **Définir Domaine** : `vtp domain ` * **Définir MDP** : `vtp password ` * **Afficher stats** : `show vtp status`
Création d'un VLAN : vlan <vlan_id> * **Attribuer un NOM** : `name `
Configuration d'une range en mode ACCESS: int range <ints> * **Mode ACCESS** : `sw mode access` ( _sw mo ac_ ) * **Attribuer le vlan aux interfaces :** `sw access vlan ` ( _sw ac vlan \_ )
  • Voir la configuration des VLAN : show vlan brief ( sh vlan br )
  • Transformer le type d'interface en mode routeur : int & no switchport (Sur SW L3 uniquement)
  • Autoriser le routage sur un switch L3 : ip routing
Créer l'interface d'un vlan : int vlan <vlan_id> * **ajouter l'IP** : `ip add `

P2.2 - Commande sur un switch CISCO

1 - RESET SWITCH

  • Se connecter au commutateur via le câble console et s'assurer que la connexion est établie.
  • Éteindre le commutateur et maintenir enfoncé le bouton Mode. Pendant que vous maintenez enfoncé le bouton Mode, allumez le commutateur. Continuez de maintenir enfoncé le bouton Mode.

Maintenez enfoncé le bouton Mode jusqu'à ce que vous voyiez la commande suivante :

flash_initboot

boot

switch:

– flash_init

– del flash:config.text

– del flash:vlan.dat

– boot

P.3 - Les ACL

P3.1 - Présentation

Les ACL, pour Access Control List, sont des règles appliquées aux trafics transitant via les interfaces du routeur que ce soit en entrée (in) ou en sortie (out). Les ACL filtrent le trafic en demandant aux interfaces d’acheminer ou non les paquets qui y transitent.

  • en : mode Privilège
  • conf t : mode Configuration
  • ip access-list standard <nom des ACL> : pour créer des ACL standards
  • show access-lists : pour montrer toutes les ACL
  • permit host : pour permettre la connexion d'une ip
  • deny any : pour ne permettre aucune autre connexion déjà rentrée
  • ip access-group <nom des ACL> in/out : le in et out dépendent de où vous voulez sécurisez

P3.2 - Création des ACL

Exemple : Appliquer l'ACL "prout" en sortie sur l'interface Fast Ethernet 0/1 :

  • en
  • conf t
  • int fastethernet0/1
  • ip access-group prout out

Exemple de création d'ACL :

ip access-list [standard|extended] [Nom Personnalisé du ACL]

  • ip access-list extended FTP-FILTER
  • tcp 192.168.10.0 0.0.0.255 any eq ftp

Possibilité d'ajouter aussi : [TCP|UDP] - (Facultatif) [eq(=equivalant)] - (Facultatif) [Nom Protocol|Port]

  • permit tcp``192.168.10.0 0.0.0.255 any``eq ftp

permit|deny : permission ou non d'accéder concernant l'adresse donnée

DIFFÉRENCE TCP/UDP : TCP est de préférence utilisé pour les communications directes nécessitant une connexion fiable (navigation Web, e-mails, SMS et transfert de fichiers). UDP sert de préférence pour la transmission de données en direct et en temps réel, lorsque la vitesse est plus importante que la fiabilité

Exemple pour bloquer un ping entre deux appareils : 

Router(config)# access-list [nom de l'ACL] deny icmp [adresse-ip qui veut bloquer] [subnet-mask] [adresse-ip qui doit être bloquée] [son subnet-mask]

P4.2 - Attribution d'un port à un VLAN

switchport mode access Définit l'interface comme interface Ethernet non agrégée à VLAN unique non balisé. Un port d'accès peut transporter du trafic dans un seul VLAN.
switchport access vlan Spécifie le VLAN pour lequel ce port d'accès transportera le trafic.
no shut Activez (activez) le port.
end Quittez le mode de configuration.

La capture d'écran suivante présente les étapes requises pour configurer le port Gigabit Ethernet (gi2) en tant que port d'accès et l'affecter à VLAN 200.

P4.3 - Configurer un port d’agrégation

switchport mode trunk Sensibiliser le numéro de port spécifié à tous les VLAN.
switchport trunk allowed vlan add Fait du port un membre dans l'ID de VLAN spécifié et lui donne une règle de sortie : Étiqueté. Cela signifie que les paquets sont balisés avec l'ID de VLAN lorsqu'ils quittent ce port sur le périphérique.
no shut Activez (activez) le port.
end Quittez le mode de configuration.

Note: En mode trunk, tous les VLAN sont autorisés par défaut. L'utilisation de la commande switchport trunk allowed vlan add vous permet de configurer les VLAN autorisés sur l'agrégation.

show interfaces switchport Affiche des informations telles que l'appartenance à un VLAN, la règle de sortie et les VLAN interdits pour le port spécifié.
